# Архитектурный обзор: Android Platform Foundation

Модуль `platform` предназначен для изоляции платформозависимых вызовов операционных систем (Android, Windows, macOS, Linux) от ядра бизнес-логики приложения ХРОНИКИ. Это позволяет ядру оставаться переносимым и тестируемым, а также упрощает сборку под различные платформы.

---

## 1. Архитектура слоев платформы

```
       [ Бизнес-ядро / Domain Layer ]
             │             │
             ▼             ▼
       [ Трейты / Platform Abstraction ]
             │             │
      ┌──────┴──────┐      └──────┐
      ▼             ▼             ▼
  [ Desktop ]   [ Android ]   [ iOS (будущее) ]
```

### Разделение по файлам в `src-tauri/src/platform/`:
- `mod.rs`: Точки входа, определение платформы на этапе компиляции.
- `notifications.rs`: Абстракции доставки системных уведомлений.
- `storage.rs`: Абстракции доступа к защищенному хранилищу ключей (Keystore/Keychain).
- `lifecycle.rs`: Отслеживание событий жизненного цикла ОС.
- `permissions.rs`: Менеджер системных прав.

---

## 2. Модель событий Жизненного Цикла (Lifecycle)

Мобильные ОС (Android) активно управляют ресурсами процесса. Приложение должно корректно реагировать на перевод в фоновый режим.

```
       [ Запуск приложения ]
                 │
                 ▼
          [ AppStarted ]
                 │
      ┌──────────┴──────────┐
      ▼ (Backgrounded)      ▼ (Resumed)
[ AppBackgrounded ] ◄─────► [ AppResumed ]
      │
      ▼ (Suspended by OS)
[ AppSuspended ]
      │
      ▼ (Killed)
  [ AppClosed ]
```

При переходе `AppBackgrounded` и `AppSuspended` все незавершенные дисковые транзакции SQLite должны завершаться, дескрипторы медиа-файлов освобождаться, а сессии безопасности блокироваться PIN-кодом (в соответствии с `SECURITY_MODEL.md`).

---

## 3. Интеграция с Android KeyStore

Мастер-пароли и ключи шифрования баз данных не хранятся в открытом виде на диске. На Android они защищаются аппаратно (StrongBox / TEE) через Android KeyStore API:

```
  [ Ввод PIN ] ──► [ PBKDF2 ] ──► [ Android KeyStore (AES-GCM Key Wrapping) ] ──► [ Расшифровка Database Key ] ──► [ SQLCipher / SQLite Open ]
```

Если устройство поддерживает биометрию, мастер-ключ может быть привязан к аутентификации по отпечатку пальца или лицу (BiometricPrompt API).
