# Архитектурное описание слоев и событий

Приложение спроектировано по принципу чистой слоистой архитектуры (Layered Architecture) с разделением ответственности и инверсией зависимостей.

---

## 1. Диаграмма слоев системы

Зависимости направлены строго сверху вниз:

```
  ┌──────────────────────────────────────────────────────────┐
  │                   Интерфейс (Svelte 5)                   │
  └────────────────────────────┬─────────────────────────────┘
                               │ (Tauri IPC / invoke)
                               ▼
  ┌──────────────────────────────────────────────────────────┐
  │                 Tauri Команды (Commands)                 │
  └────────────────────────────┬─────────────────────────────┘
                               │
                               ▼
  ┌──────────────────────────────────────────────────────────┐
  │                 Сервисы (Account/Feature)                │
  └────────────────────────────┬─────────────────────────────┘
                               │
                               ▼
  ┌──────────────────────────────────────────────────────────┐
  │                Провайдеры (AccountProvider)              │
  └────────────────────────────┬─────────────────────────────┘
                               │
                               ▼
  ┌──────────────────────────────────────────────────────────┐
  │        Интерфейсы Репозиториев (IdentityRepository)      │
  └────────────────────────────┬─────────────────────────────┘
                               │
                               ▼
  ┌──────────────────────────────────────────────────────────┐
  │            Реализация Хранилища (SQLite pool)            │
  └──────────────────────────────────────────────────────────┘
```

- **UI слой (Svelte 5)**: Отвечает за реактивный рендеринг данных. Использует `$state` и `$derived` руны для связывания интерфейса. Не знает о бэкенде или структурах БД, общается исключительно через хелпер `safeInvoke`.
- **Tauri Commands**: Промежуточный мост вызова процедур (IPC). Принимает аргументы команд от фронтенда, извлекает состояние `AppState` и пересылает вызовы в соответствующие сервисы.
- **Сервисы (Services)**: Выступают точками координации бизнес-логики. Сервисы не привязаны к конкретному источнику данных.
- **Провайдеры (Providers)**: Реализуют стратегии для конкретных механизмов (например, `LocalAccountProvider` для локальной авторизации, `SubscriptionFeatureProvider` для проверки Feature Gate подписок).
- **Интерфейсы репозиториев (Traits)**: Задают абстрактный контракт для чтения/записи данных.
- **Слой хранения (Storage)**: Реализует интерфейсы с помощью баз данных (SQLite), файлового хранилища медиа-исходников и эскизов.

---

## 2. Жизненный цикл событий: Шина событий (Event Bus)

Для развязки побочных эффектов (логирование аудита, облачная синхронизация, отправка push-уведомлений) в приложении развернута асинхронная потокобезопасная шина событий (`EventBus`) на базе `tokio::sync::broadcast`.

### Жизненный цикл события:

1. **Возникновение действия**:
   Пользователь производит операцию на клиенте (например, регистрирует аккаунт).
2. **Публикация**:
   `LocalAccountProvider` выполняет операцию сохранения данных в SQLite и вызывает метод `event_bus.publish(DomainEvent::UserRegistered { ... })`.
3. **Асинхронная доставка**:
   Событие помещается в буфер многоадресной рассылки `broadcast::channel`. Отправитель не ждет завершения обработки подписчиков.
4. **Обработка**:
   Службы-слушатели (например, планировщик синхронизации или аудит-логгер), которые ранее вызвали метод `event_bus.subscribe()`, асинхронно считывают событие из канала и выполняют свои побочные эффекты.

```
+---------------+      event_bus.publish()      +──────────────+
|  Account      | ────────────────────────────> |  EventBus    |
|  Provider     |                               |  (tokio)     |
+---------------+                               +──────────────+
                                                       │
                                        ┌──────────────┴──────────────┐
                                        │ (broadcast)                 │ (broadcast)
                                        ▼                             ▼
                              +──────────────────+          +──────────────────+
                              | Audit Log        |          | Cloud Sync       |
                              | Subscriber       |          | Subscriber       |
                              +──────────────────+          +──────────────────+
```
