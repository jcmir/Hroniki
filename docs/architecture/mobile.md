# Архитектурная спецификация: Мобильная платформа Android

Этот документ описывает принципы работы мобильной сборки приложения ХРОНИКИ под Android с использованием Tauri Mobile.

---

## 1. Стек Tauri Mobile на Android

```
       [ Webview (HTML5/TS) ] ────(Tauri IPC)────┐
                                                 ▼
                                        [ Rust Core (lib.rs) ]
                                                 │
                                           (JNI / ndk-glue)
                                                 ▼
                                       [ Android OS (Kotlin) ]
```

Rust-ядро Хроник компилируется в динамическую библиотеку (`.so`), которая подгружается Android-приложением. Tauri автоматически генерирует Activity-обертку, в которой крутится полноэкранный WebView.

---

## 2. Безопасность и Android Keystore

Для соблюдения Zero-Knowledge принципа на Android мастер-ключ базы данных шифруется ключом, хранящимся в **Android Keystore System**.
- Ключ шифрования деривируется из PIN-кода пользователя и соли с помощью Argon2id.
- Этот ключ используется для шифрования/дешифрования реального ключа SQLite (SQLCipher) через AES-GCM.
- Мета-ключ шифрования оборачивается Keystore-ключом, защищенным аппаратно на уровне TEE (Trusted Execution Environment) или StrongBox.

---

## 3. Фоновая работа и Будильники

Android жестко ограничивает работу фоновых потоков (Tokio runtime приостанавливается, когда Activity переходит в OnPause/OnStop).
Поэтому в Спринте 5.10 планировщик `ReminderScheduler` на Android будет переписан с использованием **WorkManager** или **AlarmManager** через JNI-вызовы. Это позволит системе будить приложение для показа уведомлений даже когда оно полностью закрыто.
