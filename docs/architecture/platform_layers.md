# Уровни абстракции платформы и мокирование

В этом документе описывается структура слоев платформы и правила мокирования системных вызовов для юнит- и интеграционных тестов.

---

## 1. Слои абстракции

```
[ Domain / Services ]
       │
       ▼ (Использование)
[ platform::PlatformContext ]
       │
       ├─► [ platform::adapters::desktop ] (Windows / macOS / Linux)
       └─► [ platform::adapters::android ] (Android JNI)
```

Вся бизнес-логика обращается к платформе исключительно через `PlatformContext`, который предоставляет ссылки на `Arc<dyn Trait>`.

---

## 2. Правила мокирования в тестах

Для тестирования логики работы с платформой на Desktop запрещено вызывать реальные JNI или Tauri API, которые могут отсутствовать в тестовом окружении.
Вместо этого в тестах используются моки:

### Mock Storage
Реализован в `platform::adapters::desktop::MemorySecureStoragePlatform`. Хранит секреты во внутренней `HashMap<String, Vec<u8>>` с потокобезопасной блокировкой `Mutex`.

### Mock Notifications
Реализован в `reminder::tests::MockNotificationProvider`. Перехватывает вызовы отправки и записывает их во внутренний буфер для последующих проверок (assertions).

### Mock Permissions
Возвращает `PermissionStatus::Granted` по умолчанию для всех проверок прав во время выполнения тестов, чтобы тесты не блокировались диалоговыми окнами ОС.
