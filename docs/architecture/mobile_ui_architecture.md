# Архитектура Мобильного UI ХРОНИКИ

Документ описывает архитектурные слои пользовательского интерфейса, систему реактивных сторов Svelte 5 и связь с Rust-ядром через Tauri IPC.

---

## 1. Слоевая Архитектура

```
+-------------------------------------------------------------+
| UI View Components (Timeline, EntryCard, Modals)           |
+------------------------------+------------------------------+
                               | Reactive Svelte 5 Store Subscriptions
+------------------------------v------------------------------+
| Frontend Stores (entries.ts, session.ts, platform.ts)       |
+------------------------------+------------------------------+
                               | Tauri IPC invoke() / listen()
+------------------------------v------------------------------+
| Rust Commands & Event Translators                           |
+------------------------------+------------------------------+
                               | Internal Bus
+------------------------------v------------------------------+
| Rust Domain Core & SQLite Encrypted Storage                 |
+-------------------------------------------------------------+
```

---

## 2. Назначение Сторов (`src/lib/stores/`)

### `entries.ts`
- Хранит хронологический список записей (`entries`).
- Группирует записи по датам для таймлайна.
- Предоставляет методы `loadEntries()`, `createEntry()`, `addPhoto()`.

### `session.ts`
- Управляет состоянием блокировки (`active`, `locked`, `awaitingUnlock`).
- Слушает события `session:locked` и `session:unlocked` от Rust.
- Предоставляет методы `unlockSession(pin)` и `lockSession()`.

### `platform.ts`
- Хранит информацию о способностях платформы (`PlatformCapabilities`) и разрешениях.

---

## 3. Вертикальный Срез (End-to-End Entry Creation)

1. Пользователь нажимает на FAB `+`.
2. Открывается `CreateEntryModal.svelte`.
3. Пользователь вводит заголовок, текст и выбирает фото.
4. Модальное окно вызывает `entriesStore.createEntry(...)`.
5. Стор делает IPC-вызов `invoke('create_entry', { ... })` к Rust-команде.
6. Rust сохраняет запись в зашифрованную БД SQLite, сохраняет фото в безопасное хранилище и возвращает готовый DTO.
7. Стор обновляет список записей, лента Таймлайна реактивно перерисовывается.
