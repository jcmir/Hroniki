# Паспорт проекта: ХРОНИКИ

**ХРОНИКИ** — это кроссплатформенное десктопное приложение для ведения хронологии личных объектов, событий и связанных с ними медиафайлов. Основным приоритетом продукта является абсолютная приватность, локальное хранение данных (local-first) и высокий уровень безопасности с шифрованием на стороне пользователя.

---

## 1. Основные принципы продукта

- **Локальность прежде всего (Local-First)**: Все базы данных, медиафайлы и метаданные хранятся на локальном устройстве пользователя. Приложение функционирует без необходимости в подключении к внешнему облаку.
- **Сквозная безопасность (E2EE)**: Личный PIN-код блокирует экран и защищает от физического перебора на локальном ПК, а пароль аккаунта используется для выработки криптографических ключей шифрования архивов резервных копий (схема AES-256-GCM).
- **Живые обложки и хроники объектов**: Объекты (автомобили, питомцы, растения, квартиры) представлены как живые хроники роста. Обложка объекта формируется автоматически на основе его самых последних фотографий.
- **Богатая визуальная эстетика (Rich UX)**: Интерфейс спроектирован с учетом современных гайдлайнов мобильного и десктопного дизайна — темный режим, стекломорфизм (glassmorphism), плавные переходы и физические микроанимации.

## 2. Структура каталогов проекта

```
/
├── docs/                      # Архитектурная документация (Vault знаний)
│    ├── architecture/         # Высокоуровневые слои системы
│    ├── database/             # Описание схемы данных SQLite и миграций
│    ├── decisions/            # Архитектурные решения (ADR)
│    └── modules/              # Детальное описание подсистем
│
├── src/                       # Frontend (Svelte 5 + Vite + TypeScript)
│    ├── lib/
│    │    ├── components/      # Компоненты UI (TimelineCard, CategoryIcon и др.)
│    │    ├── mock/            # Имитационные репозитории данных для тестов
│    │    ├── types/           # Объявление типов данных (Entry, TimelineEntry)
│    │    └── utils/           # Хелперы форматирования дат и времени
│    └── routes/               # Маршруты и главная страница оболочки (+page.svelte)
│
└── src-tauri/                 # Backend (Tauri + Rust + SQLite)
     ├── cargo.toml            # Декларация Rust зависимостей (sqlx, argon2, aes-gcm)
     ├── tauri.conf.json       # Конфигурация сборщика Tauri, права доступа и окна
     ├── migrations/           # Нативные SQLite-миграции базы данных (0001..0006)
     └── src/
          ├── app_state.rs     # Состояние приложения, разделяемое потоками (AppState)
          ├── main.rs          # Точка старта Tauri
          ├── lib.rs           # Инициализация пулов, служб и роутинг команд
          ├── account/         # Модуль аккаунтов (провайдеры, сессии, сервисы)
          ├── features/        # Модуль Feature Gate (подписки, Feature enums)
          ├── events/          # Модуль шины событий (EventBus, tokio-broadcast)
          ├── storage/         # Хранилище (SQLite, миграции, репозитории)
          ├── commands/        # Команды Tauri для связи с Frontend
          └── security/        # Модуль криптографии (шифрование, пароли, Argon2)
```

## 3. Требования к сборке и запуску

Для сборки проекта на локальной машине разработчика требуются:
- Node.js (версии 18 и выше)
- Rust & Cargo (тулчейн Rustup)
- Tauri CLI (`npm install -g @tauri-apps/cli`)

### Разработка
1. Установите зависимости:
   ```bash
   npm install
   ```
2. Запустите приложение в режиме отладки:
   ```bash
   npm run tauri dev
   ```

### Тестирование бэкенда
Для запуска всех Rust тестов (Unit, интеграционные и криптографические):
```bash
cargo test
```

### Сборка продуктового инсталлятора
Для сборки Windows-установщика:
```bash
npm run tauri build
```
