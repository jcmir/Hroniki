# Спецификация безопасности: Криптографическая корректность AES-GCM

Этот документ описывает требования к реализации AEAD-шифрования и защиты целостности данных в приложении ХРОНИКИ.

---

## 1. Схема AEAD шифрования (AES-256-GCM)

Мы применяем схему **Authenticated Encryption with Associated Data** (AEAD) с использованием алгоритма AES-256 в режиме GCM (Galois/Counter Mode).

```
   Plaintext ───► [ AES-256-GCM Encryption ] ◄─── Master Key (MK)
                             │
                             ├──────────────────────────┐
                             ▼                          ▼
                        Ciphertext            Authentication Tag (16 bytes)
```

### Защита целостности:
- При дешифровании алгоритм использует `Authentication Tag` (тег аутентификации) для валидации неизменности шифротекста и вектора инициализации (`nonce`).
- Любое изменение хотя бы одного бита в полях `ciphertext`, `tag` или `nonce` гарантированно приводит к ошибке дешифрования `KeyStoreError::DecryptionFailed`.

---

## 2. Изоляция ошибок (Error Privacy)
Во избежание атак по сторонним каналам (Side-channel attacks) и утечек информации о криптосистеме, все ошибки декодирования или несовпадения тегов возвращают унифицированную ошибку `KeyStoreError::DecryptionFailed`, не раскрывая подробности внутреннего состояния Cipher.
