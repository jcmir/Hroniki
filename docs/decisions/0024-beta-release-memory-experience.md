# ADR 0024: Архитектура Бета-Релиза, Навигация Объектов и Версионирование Резервных Копий

## Контекст и Проблема
После реализации таксономии объектов и поиска в Спринте 5.17 необходимо подготовить приложение ХРОНИКИ к первому выпуску Бета-версии для пользователей. Требуется внедрение выделенной маршрутизации просмотра истории объекта (`/object/[id]`), жизненного цикла первого запуска (`FirstRun` ➔ `Setup` ➔ `Ready`), структурированной модели настроек `UserSettings` и версионирования резервных копий через файл `manifest.json`.

## Решения

### 1. Навигация Просмотра Истории Объекта (`/object/[id]`)
Вместо изолированного модального окна используется полноценный SvelteKit маршрут `src/routes/object/[id]/+page.svelte`. Это обеспечивает поддержку диплинков (`hroniki://object/<id>`), корректную обработку кнопки «Назад» на Android и сохранение состояния хроники объекта.

### 2. Версионирование Резервных Копий (`manifest.json`)
Все резервные копии zip-архивов включают обязательный файл метаданных `manifest.json`:
```json
{
  "version": "1",
  "app": "hroniki",
  "created_at": "2026-07-20T00:00:00Z"
}
```
При импорте резервной копии происходит считывание манифеста и проверка версии формата данных перед восстановлением SQLite базы и медиа-файлов.

### 3. Структурированная Модель Настроек (`UserSettings`)
Параметры пользователя группируются в сущность `UserSettings`:
- `profile_name`: Имя пользователя.
- `theme`: Тема оформления (`System`, `Dark`, `Light`).
- `auto_lock_minutes`: Тайм-аут автоблокировки.
- `notifications`: Флаг уведомлений.
- `biometric_unlock`: Флаг биометрии.
