# ADR 0020: Мобильный UI Shell, Архитектура Сторов и Вертикальный Срез Создания Записей

## Контекст и Проблема
После завершения Спринтов 5.12 и 5.13 платформа ХРОНИКИ имеет надежный слой мобильного runtime, нативные Kotlin-мосты и автомат состояний сессии. Однако для превращения инфраструктуры в рабочий пользовательский продукт необходим пользовательский интерфейс (UI Shell) на Svelte 5, обеспечивающий вертикальный срез: создание хронологической записи с вложением фото, её сохранение в зашифрованную БД SQLite и отображение на Таймлайне.

При этом UI не должен напрямую зависеть от внутренних структур Rust домена или событий `DomainEvent`, чтобы не создавать жесткой связанности.

## Архитектура Слоев (IPC Isolation Boundary)

```
┌────────────────────────────────────────────────────────┐
│                   UI Components                        │
│ (Timeline.svelte, EntryCard.svelte, CreateEntryModal)  │
└───────────────────────────▲────────────────────────────┘
                            │ Reactive Subscribe
┌───────────────────────────┴────────────────────────────┐
│                  Frontend Stores                       │
│      (entries.ts, session.ts, platform.ts)             │
└───────────────────────────▲────────────────────────────┘
                            │ IPC Invoke / Event Listeners
┌───────────────────────────┴────────────────────────────┐
│                    Tauri IPC Bridge                    │
└───────────────────────────▲────────────────────────────┘
                            │ Domain Events & Commands
┌───────────────────────────┴────────────────────────────┐
│                   Rust Domain Core                     │
└────────────────────────────────────────────────────────┘
```

## Рассматриваемые Варианты

1. **Прямое подписывание Svelte-компонентов на DomainEvents и Tauri IPC**:
   - *Плюсы*: Меньше кода в начале.
   - *Минусы*: Нарушение разделения ответственности, дублирование кода обработки ошибок IPC в каждом компоненте, хрупкость при рефакторинге UI.

2. **Svelte Stores как IPC-граница (Выбран)**:
   - *Плюсы*: Полная изоляция UI от IPC вызовов. Компоненты реагируют на изменения реактивных сторов (`sessionStore`, `entriesStore`), а сторы централизованно инвоцируют Tauri IPC команды и слушают события Rust.

## Эстетика Интерфейса
Интерфейс оформляется в стиле современного личного дневника воспоминаний (по аналогии с Apple Photos и Day One Journal):
- Выдержанная палитра темной темы (Deep Obsidian #0D0F12, карточки Slate #1A1D24, акценты Warm Amber #F59E0B).
- Большое количество воздуха, четкая типографика, выделение дат и крупное отображение фотографий.

## Решение
Принято решение реализовать:
- Набор сторов Svelte 5 в `src/lib/stores/` (`session.ts`, `entries.ts`, `platform.ts`).
- Компоненты интерфейса: `EntryCard.svelte`, `CreateEntryModal.svelte`, `PINLockModal.svelte`.
- Главную страницу `src/routes/+page.svelte` со встроенным сквозным потоком добавления записей и фото.
