# ADR 0013: Изоляция криптографических ключей Android Keystore от Rust Core

## Контекст и Проблема
При интеграции мобильного хранилища секретов (Android Keystore / iOS Keychain) существует риск утечки криптографического материала во временную память (RAM) приложения. Rust Core не должен владеть сырыми байтами мастер-ключа шифрования во избежание скомпрометирования при эксплоитах.

## Решение
Мы фиксируем следующие архитектурные правила:
1. **Rust never sees the Master Key**: Rust-слой никогда не видит и не содержит сырых байт мастер-ключа шифрования.
2. **Делегация шифрования нативной платформе**: Вся операция шифрования (wrapping) и дешифрования (unwrapping) выполняется внутри Kotlin/Android OS с использованием защищенного чипа TEE (Trusted Execution Environment) или StrongBox.
3. **Обмен сериализованными WrappedSecret**: Rust передает нативной стороне сырые байты данных, а получает обратно готовый сериализованный JSON/DTO `WrappedSecret` (с nonce, ciphertext и tag), сгенерированные и проверенные нативным кодом.

## Последствия
- Полная аппаратная защита ключей БД и сессий в памяти TEE.
- Чистые границы JNI: Rust оперирует лишь абстрактными контрактами `KeyStoreBackend`.
