# ADR 0025: Публичный Идентификатор Выпуска, DemoExperienceService и Регламент Бета-Тестирования

## Контекст и Проблема
Приложение ХРОНИКИ вышло на этап Бета-тестирования на физических Android устройствах. Необходим переход на публичный мобильный идентификатор пакета (`app.hroniki.mobile`), реализация сервиса мастер-старта `DemoExperienceService`, выделенного экрана «Центр Памяти» (`/reminders`) и регламента сбора фидбека первых тестеров.

## Идентификатор Пакетного Выпуска
Установлен мобильный идентификатор приложения:
- **Application ID**: `app.hroniki.mobile`
- **Version Name**: `0.2.1-beta`
- **Version Code**: `21`

## Спецификация `DemoExperienceService`
Модуль управляет безопасной инициализацией демо-данных при первом старте:
- `has_seen_demo()`: Флаг просмотра демо-мастера.
- `offer_demo()`: Запрос разрешения у пользователя на сидирование стартовых объектов.
- `seed_demo()`: Создание 4 стартовых объектов (`🚗 BMW X5`, `🏡 Дом в Завидово`, `🌱 Сад`, `✈️ Поездка на Алтай`) с готовыми хрониками и фотографиями.

## Регламент Сбора Обратной Связи (Beta Feedback Loop)
Бета-тестеры отправляют отчеты по форме:
- **Устройство / ОСР**: Модель телефона и версия Android (напр. Pixel 7 / API 34).
- **Сценарий**: Нажатое действие (первичная регистрация, добавление фото, бэкап).
- **Скриншот / Лог**: Приложенный скриншот интерфейса.
