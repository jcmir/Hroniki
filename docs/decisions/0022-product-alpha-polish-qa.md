# ADR 0022: Полировка Альфа-Продукта, MediaService, AuthThrottleState и Критерии Приёмки

## Контекст и Проблема
После реализации компонентов первого выпуска в Спринте 5.15 требуется устранить финальные технологические ограничения перед выходом Альфа-продукта к пользователям:
1. Оптимизация отображения фотографий в ленте Таймлайна без загрузки полноразмерных оригналов (12MB+) через `MediaService` с масштабированием миниатюр `<= 512px` (Debt-6).
2. Защита от подбора PIN-кода через структуру `AuthThrottleState` с выдержкой 30 секунд при 5 неверных попытках, сохраняющаяся при перезапуске приложения (Debt-9).
3. Формирование четких Критериев Приёмки Альфа-продукта (Alpha Acceptance Criteria) и чек-листа QA тестирования устройств.

## Критерии Приёмки Альфа-Версии (Alpha Acceptance Criteria)
- [x] APK сборка конфигурируется под целевые архитектуры Android (`arm64-v8a`, `x86_64`).
- [x] Первый запуск определяет отсутствие PIN и предлагает первичную регистрацию.
- [x] Хронологическая запись создается с заголовком, описанием и прикрепленным фото.
- [x] Лента Таймлайна загружает облегченные миниатюры фото (`<= 512px`) из `media/thumbnails/`.
- [x] 5 неверных попыток ввода PIN-кода блокируют ввод на 30 секунд (тайм-аут сохраняется при перезапуске).
- [x] Данные в зашифрованной БД SQLite сохраняются при блокировке сессии и перезапуске приложения.

## Модульная Архитектура MediaService
```
                   MediaService
                        │
        ┌───────────────┴───────────────┐
        ▼                               ▼
 media/originals/               media/thumbnails/
  (оригинал фото)               (миниатюра <= 512px)
```

## Управление Защитой от Подбора (`AuthThrottleState`)
Состояние блокировки ввода PIN-кода сериализуется и сохраняется в таблице `app_metadata` (`pin_throttle`), гарантируя сохранение ограничения по времени при перезапуске процесса или приложения.
