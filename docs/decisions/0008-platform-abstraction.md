# ADR 0008: Абстракция платформы и адаптеров

## Контекст и Проблема
Приложение ХРОНИКИ переходит на кроссплатформенную мобильную архитектуру (Desktop + Android). Реализация системных сервисов (хранилище ключей, уведомления, права доступа) кардинально отличается на разных ОС. Использование макросов условной компиляции `#[cfg(target_os = "android")]` непосредственно в логике домена или сервисов делает код нечитаемым, хрупким и непригодным для тестирования на Desktop.

## Решение
Мы внедряем **Platform Layer** на базе паттерна Адаптер (Ports and Adapters):
1. **Порты (Traits)**: Определяются в `platform/` (`NotificationPlatform`, `SecureStoragePlatform`, `PermissionPlatform`).
2. **Контекст (`PlatformContext`)**: Объединяет все порты в единый потокобезопасный реестр `PlatformContext`, доступный через `AppState`.
3. **Адаптеры**: Разделяются по файлам в подкаталоге `platform/adapters/`. 
   - `desktop.rs`: содержит реализации для настольных ОС.
   - `android.rs`: содержит JNI-заглушки для Android сборки.
4. **Безопасные пространства имен**: Трейт `SecureStoragePlatform` оперирует структурой `SecretIdentifier { kind, namespace }` для предотвращения пересечения ключей и обеспечения будущей изоляции архивов/пользователей.

## Последствия
- Полная изоляция платформозависимого кода. Бизнес-логика Хроник свободна от `#[cfg]`.
- Тестирование мобильной логики на Desktop через Memory/Mock адаптеры.
- Подготовлен фундамент для беспрепятственного запуска Android билда.
