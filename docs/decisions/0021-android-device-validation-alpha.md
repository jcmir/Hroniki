# ADR 0021: Валидация Android Alpha, Scoped Storage Фото-Пайплайн и Argon2id SessionVerifier

## Контекст и Проблема
После реализации мобильного UI Shell в Спринте 5.14 необходимо обеспечить готовность первого релизного альфа-пакета (`hroniki.apk`), закрыв четыре ключевых продуктовых требования:
1. Валидация сборки APK на реальных мобильных архитектурах (`arm64-v8a`, `x86_64`) без утечки путей внешней платформы (Debt-5).
2. Безопасный пайплайн обработки фотографий устройств Android 10+ (Scoped Storage / SAF `content://` URI) с категоризацией источников `MediaSource` (`Camera`, `Gallery`, `ImportedFile`) (Debt-6).
3. Криптографическая аутентификация сессии через структуру `SessionVerifier` с использованием Argon2id (Debt-7).
4. Опыт первого запуска (First Launch Experience): регистрация первичного PIN-кода при отсутствии заданного секрета (Debt-8).

## Решение

### 1. Argon2id SessionVerifier (Debt-7 & Debt-8)
Для проверки владения секретом без его хранения в открытом виде создается структура `SessionVerifier`:
```rust
pub struct SessionVerifier {
    pub salt: String,
    pub nonce: String,
    pub ciphertext: Vec<u8>,
    pub tag: Vec<u8>,
}
```
При первом запуске пользователя запрашивается установка PIN-кода, из которого с помощью Argon2id выводится промежуточный ключ, зашифровывающий контрольный маркер сессии. При повторных запусках ввод правильного PIN-кода расшифровывает маркер и восстанавливает оперативную сессию. Ввод неверного PIN-кода отклоняется, оставляя состояние сессии заблокированным (`SessionState::Locked`).

### 2. Scoped Storage Safe Media Pipeline (Debt-6)
Файлы фотографий, передаваемые через Android Photo Picker или файловый диалог, потребляются в виде байтовых потоков через Kotlin Bridge / IPC, сохраняются в песочнице `media/originals/` с присвоением категории `MediaSource` (`Camera`, `Gallery`, `ImportedFile`). При отсутствии физического файла на диске UI корректно отображает индикатор ошибки без сбоев интерфейса.

### 3. Build & Packaging Gate (Debt-5)
Сборка Android таргетов настраивается с поддержкой `arm64-v8a` и `x86_64`, гарантируя успешное компилирование ABI и отсутствие зависимости домена от Android SDK.
