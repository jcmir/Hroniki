# Спецификация модуля: Product Alpha

Модуль `product_alpha` определяет правила и инварианты функционирования Альфа-версии приложения ХРОНИКИ.

---

## 1. Сценарии Первого Запуска (First Launch Experience)

```
[Старт приложения]
        │
        ├─► PIN настроен? ──► Да ──► [Показать PINLockModal (Ввод PIN)]
        │
        └─► Нет ───────────────────► [Показать PINLockModal (Первичная установка PIN)]
                                                   │
                                                   ▼
                                     [Создание SessionVerifier (Argon2id)]
                                                   │
                                                   ▼
                                        [Переход на чистый Таймлайн]
```

---

## 2. Источники МедиаДанных (`MediaSource`)

Вложения фотографий к хроническим событиям классифицируются через `MediaSource`:
- `Camera`: Прямой снимок с камеры мобильного устройства.
- `Gallery`: Выбор из галереи изображений (через Scoped Storage / Photo Picker).
- `ImportedFile`: Импортированный файл из хранилища.

Каждое вложение безопасно копируется в изолированное каталожное хранилище приложения `media/originals/` и привязывается к ID записи в БД SQLite.

---

## 3. Инварианты Безопасности

- Ввод неверного PIN-кода 5 раз подряд блокирует повторный ввод на 30 секунд.
- Очистка оперативной памяти сессии при блокировке приложения (`ApplicationLocked`) сохраняет зашифрованную базу данных SQLite на диске без изменений.
