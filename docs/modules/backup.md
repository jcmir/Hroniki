# Документация модуля: Backup & Restore (Резервное копирование)

Модуль `backup` отвечает за создание переносимых локальных зашифрованных контейнеров бэкапа (.hroniki) и восстановление из них всей истории данных с контролем версий и совместимости.

---

## 1. Назначение

- Упаковка базы данных SQLite и папки оригинальных медиафайлов в единый переносимый архив.
- Шифрование контейнера по схеме PBKDF2 + AES-256-GCM с защитой паролем.
- Контроль совместимости схем базы данных при импорте для предотвращения сбоев и даунгрейдов.
- Автоматическая миграция импортированных баз данных под текущую версию приложения.

---

## 2. Спецификация формата архива (Archive Format v2)

Формат `.hroniki` представляет собой ZIP-архив, зашифрованный с помощью алгоритма AES-256-GCM.
Внутри расшифрованного ZIP-архива содержатся следующие файлы:

```
├── manifest.json         # Метаданные архива v2 (версии, метки экспорта)
├── chronology.sqlite     # Снимок базы данных SQLite (снапшот VACUUM INTO)
└── media/                # Оригинальные медиафайлы событий хроники
     ├── photo_uuid1.jpg
     └── photo_uuid2.png
```

### Структура `manifest.json`:
```json
{
  "app_name": "ХРОНИКИ",
  "version": "1.0.0",
  "schema_version": 6,
  "encryption_version": "AES-256-GCM",
  "exported_at": "2026-07-19T18:45:00Z"
}
```

---

## 3. Логика контроля совместимости при импорте

```
            Импорт архива
                  │
                  ▼
          [ Расшифровка ]
                  │
                  ▼
         Есть manifest.json?
             /        \
          Да           Нет (v1 архив)
          /             \
m.schema_version > 6?     Разрешить импорт
      /       \
   Да          Нет
   /             \
Заблокировать  Разрешить импорт
импорт         & запустить migrations
```

1. **Проверка манифеста**: Если файл `manifest.json` найден, система считывает поле `schema_version`.
2. **Блокировка даунгрейда**: Если версия схемы в архиве больше текущей поддерживаемой версии схемы бэкенда (например, `schema_version` > `6`), импорт немедленно отклоняется с ошибкой во избежание сбоев.
3. **Обратная совместимость**: Если манифест отсутствует, архив считается версией v1 и успешно распаковывается.
4. **Горячая миграция**: После распаковки файлов бэкенд пересоздает `SqlitePool` соединение, запускает цикл миграций и обновляет ссылки в глобальном `AppState` без остановки или сбоев приложения.

---

## 4. Public API

Tauri-команды экспорта и импорта:

```rust
// Экспортировать базу и медиафайлы в файл .hroniki
#[tauri::command]
pub async fn export_archive(
    password: String,
    state: State<'_, AppState>,
    app: tauri::AppHandle
) -> Result<String, String>;

// Импортировать историю данных и медиафайлы из файла .hroniki
#[tauri::command]
pub async fn import_archive(
    password: String,
    state: State<'_, AppState>,
    app: tauri::AppHandle
) -> Result<(), String>;
```
