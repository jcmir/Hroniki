# Документация модуля: Audit Log (Аудит)

Модуль `audit` отвечает за журналирование действий пользователей и критических системных событий безопасности. Реализован как асинхронный независимый подписчик событий домена.

---

## 1. Назначение

- Фиксация фактов успешной и неуспешной аутентификации.
- Запись событий экспорта и импорта резервных копий.
- Логирование изменения тарифных планов пользователей.
- Предоставление защищенного реестра действий для целей аудита (Enterprise Foundation).

---

## 2. Основные сущности

- **`AuditLogEntry`**: Запись журнала аудита. Содержит:
  - `id`: Уникальный UUID записи.
  - `user_id`: Уникальный UUID пользователя (опционально, так как события входа/регистрации могут происходить до авторизации).
  - `event_type`: Текстовое имя события (например, `UserAuthenticated`, `ArchiveExported`).
  - `details`: Сериализованный в JSON объект с деталями события (например, имя экспортированного файла, статус входа).
  - `created_at`: Метка времени записи в формате UTC.

---

## 3. Архитектура интеграции с EventBus

Логирование осуществляется полностью реактивно через подписку на шину `EventBus`:

1. Провайдеры и службы публикуют доменные события (`DomainEvent`) в `EventBus`.
2. Фоновый подписчик `AuditSubscriber` считывает события из канала вещания `tokio::sync::broadcast`.
3. Полученные события преобразуются в записи `AuditLogEntry`.
4. Записи отправляются в `AuditRepository` для сохранения в SQLite.

---

## 4. Public API

Интерфейс `AuditService` координирует запись и чтение журналов:

```rust
impl AuditService {
    // Получить историю аудита для конкретного пользователя (или все системные события)
    pub async fn get_user_logs(
        &self,
        user_id: Option<&str>
    ) -> Result<Vec<AuditLogEntry>, IdentityError>;

    // Записать лог напрямую (используется подписчиком)
    pub async fn record_event(
        &self,
        user_id: Option<String>,
        event_type: &str,
        details: Option<String>
    ) -> Result<(), IdentityError>;
}
```

---

## 5. Будущие изменения

- Шифрование записей таблицы `audit_logs` на лету (Encrypted Audit Trail) для защиты логов от просмотра посторонними лицами с прямым доступом к базе данных.
- Экспорт логов аудита в формате подписанного JSON (Signed Audit Log) для гарантии невозможности подделки записей.
