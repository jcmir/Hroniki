# Документация модуля: Security (Безопасность)

Модуль `security` объединяет криптографические операции и политики физической защиты данных пользователя. Вся безопасность спроектирована по принципу нулевого доверия (Zero-Knowledge) — пароли и ключи не хранятся в открытом виде и известны только пользователю.

---

## 1. Защита PIN-кодом (Физический Lock Screen)

Для защиты запущенного приложения от доступа третьих лиц реализован экран блокировки (PIN Lock Screen):
- **Локальный PIN-код**: Задается при онбординге. Хешируется по схеме `PBKDF2` (100 000 итераций + соль) на локальном устройстве.
- **Защита от перебора (Brute-Force Lockout)**:
  - Допускается максимум **5 неверных попыток ввода**.
  - Счетчик неудачных попыток сохраняется в оперативной памяти.
  - При достижении лимита включается кулдаун-таймер на **15 секунд**. В это время клавиатура полностью блокируется, а на экране выводится мягкое предупреждение («Попробуйте снова через 15 секунд») с анимированной шкалой отсчета.

---

## 2. Хеширование паролей аккаунтов (Argon2id)

Для защиты учетных записей при будущей синхронизации используется стандарт `Argon2id` (версия 0.5):
- Пароль пользователя хешируется со случайной солью, генерируемой криптографически стойким ГПСЧ (`rand::rngs::OsRng`).
- Параметры Argon2id выбраны с учетом баланса производительности и устойчивости к перебору на видеокартах (GPU cracking resistance).
- База данных хранит только результирующую строку хеша, пароль в открытом виде никогда не попадает в оперативную память на длительное время.

---

## 3. Шифрование архивов данных (AES-256-GCM)

Служба резервного копирования использует симметричное шифрование для контейнеров бэкапа:
- **Выработка ключа (KDF)**: Из введенного пользователем пароля с помощью алгоритма `PBKDF2-HMAC-SHA256` вырабатывается 256-битный симметричный ключ шифрования.
- **Шифрование контейнера**: Данные сжимаются в ZIP-буфер и зашифровываются алгоритмом `AES-256-GCM` с аутентификацией данных.
- **Контроль целостности**: Использование режима `GCM` гарантирует обнаружение любых попыток модификации или повреждения байт зашифрованного архива до начала его распаковки.

---

## 4. Используемые зависимости в Cargo

Модули безопасности опираются на надежные Rust библиотеки:
- `aes-gcm = "0.10"` — криптография AES-GCM.
- `pbkdf2 = "0.12"` — выработка ключа из пароля и PIN.
- `argon2 = "0.5"` — хеширование паролей учетных записей.
- `rand = "0.8"` — генерация криптостойких солей и векторов инициализации (IV).
