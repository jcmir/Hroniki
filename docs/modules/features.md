# Документация модуля: Feature Gate (Features)

Модуль `features` отвечает за разграничение прав доступа и коммерческую монетизацию. Спроектирован по принципу разделения тарифов и конкретных функциональных флагов (Feature).

---

## 1. Назначение

- Разграничение доступа к продвинутым функциям (ИИ, облачный бэкап, семейные архивы).
- Динамическое включение/выключение возможностей без изменения кода бизнес-логики.
- Подготовка фундамента для тарифных планов подписки.

---

## 2. Основные сущности

- **`Feature`** (enum): Перечисление функциональных возможностей приложения:
  - `AI`: Доступ к поиску и RAG-моделям.
  - `CloudSync`: Синхронизация между устройствами.
  - `FamilyArchive`: Совместные семейные хроники.
  - `UnlimitedObjects`: Создание неограниченного числа объектов (на Free тарифе действует лимит).
  - `ExportBackup`: Локальный бэкап архива данных.
- **`SubscriptionPlan`** (enum): Уровни подписок пользователей:
  - `Free`: Базовый бесплатный тариф. Разрешен только экспорт бэкапов.
  - `Pro`: Индивидуальный премиум-тариф. Включает неограниченное число объектов, ИИ-функции и облачную синхронизацию.
  - `Family`: Семейный тариф. Дополнительно включает общий доступ к семейным архивам.
  - `Enterprise`: Корпоративный тариф.

---

## 3. Public API

Интерфейс `FeatureService` координирует проверки доступности возможностей:

```rust
impl FeatureService {
    // Проверить, активна ли конкретная функция для пользователя
    pub async fn is_enabled(
        &self,
        user_id: &str,
        feature: Feature
    ) -> bool;

    // Получить полный список активных функций пользователя
    pub async fn get_enabled_features(
        &self,
        user_id: &str
    ) -> Vec<Feature>;

    // Запросить текущий тариф пользователя
    pub async fn get_user_plan(
        &self,
        user_id: &str
    ) -> Result<SubscriptionPlan, IdentityError>;

    // Обновить тарифный план пользователя (апгрейд подписки)
    pub async fn update_user_plan(
        &self,
        user_id: &str,
        plan: SubscriptionPlan,
    ) -> Result<(), IdentityError>;
}
```

---

## 4. Используемые сервисы

- `SubscriptionRepository`: Для получения и изменения активных тарифов из базы данных SQLite.
- `FeatureProvider`: Трейт-провайдер, определяющий политику сопоставления тарифов со списком включенных функций.

---

## 5. Будущие изменения

- Вынесение политик прав из кода в отдельный файл конфигурации или слой `FeaturePolicy` для динамического добавления новых тарифов и промо-периодов.
- Интеграция с платежными шлюзами (Stripe, App Store, Google Play) на уровне внешнего провайдера подписок.
