# UX Bug Checklist — HRONIKI Beta 0.2.1

Проверяется на реальном устройстве Android 13+ (API 33+) и эмуляторе Pixel 7 API 34.

---

## 1. Клавиатура

| # | Проверка | Ожидаемое | ✓/✗ |
|---|---|---|---|
| K1 | Открыть форму создания записи | Поля ввода видны | |
| K2 | Нажать на поле «Заголовок» | Клавиатура появляется, поле НЕ перекрыто | |
| K3 | Клавиатура скрывает кнопку «Сохранить» | Кнопка доступна через scroll | |
| K4 | Нажать «Назад» | Клавиатура скрывается, форма остаётся | |

**Решение при сбое**: `android:windowSoftInputMode="adjustResize"` в `AndroidManifest.xml`.

---

## 2. FAB и Navigation Bar

| # | Проверка | Ожидаемое | ✓/✗ |
|---|---|---|---|
| F1 | На главном экране | FAB (фиолетовый «+») полностью виден | |
| F2 | Жест navigation bar (Android 10+) | FAB не перекрыт navbar | |
| F3 | FAB нажимается | Открывает форму создания | |

**Решение при сбое**: добавить `paddingBottom` = `navigationBarHeight` через WindowInsets.

---

## 3. Safe Area Insets (Edge-to-Edge)

| # | Проверка | Ожидаемое | ✓/✗ |
|---|---|---|---|
| S1 | Status bar область | Контент не рисуется под status bar | |
| S2 | Навигационный бар (жестовая навигация) | Нижний контент не обрезан | |
| S3 | Android 15: system bars area | Контент сдвинут ниже / использует insets | |
| S4 | На телефоне с вырезом (notch) | Хроники-заголовок не обрезан | |

---

## 4. Status Bar Color

| # | Проверка | Ожидаемое | ✓/✗ |
|---|---|---|---|
| C1 | Открыть приложение | Status bar иконки тёмные (на светлом фоне) | |
| C2 | Цвет status bar | Соответствует `#FCF7FA` или прозрачный | |

**Решение при сбое**: `WindowInsetsControllerCompat.isAppearanceLightStatusBars = true`.

---

## 5. Dark Mode

| # | Проверка | Ожидаемое | ✓/✗ |
|---|---|---|---|
| D1 | Включить Dark Mode на телефоне | ХРОНИКИ остаётся в своей цветовой схеме | |
| D2 | `forceDarkAllowed="false"` проверка | Фон `#FCF7FA`, НЕ инвертированный | |
| D3 | Фиолетовый акцент | Сохраняется `#7C3AED` | |
| D4 | GlassCard | Остаётся `rgba(255,255,255,0.72)`, не чёрным | |

> [!CAUTION]
> Без `android:forceDarkAllowed="false"` Android может автоматически инвертировать весь UI. Это уже добавлено в AndroidManifest.xml Sprint 5.20.
