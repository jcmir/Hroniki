# Android Alpha Device QA Checklist

Чек-лист проверки качества и сценариев тестирования Альфа-версии приложения ХРОНИКИ на мобильных устройствах и эмуляторах (Pixel 7 / Android API 34).

---

## Чек-лист Проверок

- [ ] **1. Чистая Установка (Fresh Install)**:
  - [ ] Приложение запускается без ошибок.
  - [ ] Отображается экран `PINLockModal` в режиме "Первичная установка PIN-кода".
  - [ ] Задание 4-значного PIN-кода открывает главный Таймлайн.

- [ ] **2. Вертикальный Срез Создания Записи**:
  - [ ] Нажатие на плавающую кнопку `+` открывает модальное окно `CreateEntryModal`.
  - [ ] Заполнение заголовка, текста и прикрепление фото.
  - [ ] Сохранение создает карточку в ленте Таймлайна.
  - [ ] Миниатюра фотографий отображается корректно и быстро.

- [ ] **3. Защита и Блокировка Сессии**:
  - [ ] Нажатие на кнопку 🔒 в верхней панели блокирует приложение.
  - [ ] Ввод неверного PIN-кода отображает сообщение об ошибке.
  - [ ] Ввод неверного PIN-кода 5 раз подряд включат блокировку на 30 секунд.
  - [ ] Перезапуск приложения во время блокировки сохраняет таймер отсчета.
  - [ ] Ввод корректного PIN-кода разблокирует сессию и восстанавливает Таймлайн.

- [ ] **4. Сохранность Данных (Data Persistence)**:
  - [ ] Перезапуск приложения сохраняет ранее созданные записи и медиа-файлы.
