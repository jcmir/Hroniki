## 1. Миссия

> **ХРОНИКИ** — это приватная, зашифрованная хроника жизни для людей и организаций, которым важно помнить всё и контролировать, кто имеет доступ к их истории.

ХРОНИКИ — не дневник и не база данных. Это **структурированная память** о событиях вокруг объектов: людей, животных, недвижимости, бизнеса, здоровья. Пользователь создаёт объекты (например, «Яблоня в саду», «Мой автомобиль», «Бабушка») и ведёт хронику событий вокруг каждого из них.

## 3. Целевые пользователи

### 3.1 Personal User
Человек, которому важна история собственной жизни:
- ведёт записи о здоровье семьи и питомцев;
- документирует ремонт, сад, автомобиль;
- хочет хранить всё локально и зашифровано.

### 3.2 Power User
Предприниматель, менеджер, специалист:
- ведёт хронику по клиентам, проектам, сотрудникам;
- нужен быстрый контекстный поиск;
- критически важна приватность данных.

### 3.3 Family
Семья, которая ведёт общий архив:
- несколько человек с разными ролями;
- общий доступ к части записей;
- история рода, семейный архив.

### 3.4 Enterprise User
Организация с требованиями к соответствию:
- аудит действий пользователей;
- шифрование данных на уровне политик;
- управление устройствами и сессиями.

## 4. Ключевые сценарии использования

### Сценарий 1: «Хроника питомца»
Пользователь создаёт объект «Кот Барсик». Добавляет записи: «Вакцинация от бешенства», «Начал прихрамывать», «Визит к ветеринару — артрит». Через год при следующем визите к ветеринару достаёт полную хронику одним поиском.

### Сценарий 2: «Семейный дом»
Объект «Дача в Подмосковье». Записи: «Положили новый фундамент», «Установили солнечные панели», «Сосед затопил — протокол». Полная история объекта за 10 лет.

### Сценарий 3: «Деловой архив»
Объект «Клиент — Иван Петров». Записи: «Первый контакт», «Отправлено КП», «Подписан договор», «Жалоба на качество — решена». Контекст для любого разговора.

### Сценарий 4: «AI-ассистент памяти» (Phase 2)
Пользователь спрашивает: «Когда последний раз менял масло в машине?» — AI находит релевантную запись из хроники объекта «Ford Focus» с точной датой и деталями.

## 6. AI и RAG-стратегия (Phase 2)

### Архитектура
```
Записи хроники
     │
     ▼
Embedding Model (локальная — Ollama / remote — OpenAI)
     │
     ▼
Vector Store (SQLite + sqlite-vss или Qdrant embedded)
     │
     ▼
RAG Engine
     │
     ▼
AI-ответ с контекстом из хроники
```

### Принципы
- **Privacy-first**: по умолчанию локальная модель.
- **Opt-in**: пользователь выбирает, использовать ли облачный AI.
- **Прозрачность**: AI всегда показывает источник — ссылку на запись.

| Тариф | Цена (планируемая) | Модель |
|-------|--------------------|--------|
| Free | $0 | Freemium — ограничения по объектам |
| Pro | ~$5–8/мес | Подписка |
| Family | ~$12–15/мес | Подписка (до 6 членов) |
| Enterprise | По запросу | Лицензия + SLA |

### Ключевые принципы
- Нет рекламы.
- Нет продажи данных.
- Данные пользователя принадлежат только пользователю.
- Приложение работает без интернета (offline-first).

## 8. Безопасность (сводка)

> Полная модель: см. [`docs/security/SECURITY_MODEL.md`](../security/SECURITY_MODEL.md)

- Шифрование AES-256-GCM для всех экспортируемых архивов.
- Argon2id для паролей пользователей.
- PBKDF2 для PIN-кода.
- Recovery Keys для восстановления доступа.
- Аудит-лог всех критических действий (Enterprise).
- Zero-knowledge design: сервер не имеет доступа к незашифрованным данным.
